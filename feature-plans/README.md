# Feature Plans

This directory contains planning artifacts generated by the `feature-planning` skill.

## Purpose

These are working documents produced during interactive feature planning sessions. They are used as input for the `feature-implementation` skill.

## Important Note

**These documents are NOT product documentation.**

They are planning artifacts that capture requirements, constraints, and verification criteria for features being developed. Once a feature is implemented, the plan document may be archived or removed.

## Structure

Each plan document (`FEATURE_NAME.md`) contains:

- **Section 1:** Feature name and description
- **Section 2:** Feature requirements
- **Section 3:** Constraints
- **Section 4:** Feature verification testing

## Workflow

1. Run `/feature-planning` to create a new plan
2. Run `/feature-implementation` to implement from a plan
